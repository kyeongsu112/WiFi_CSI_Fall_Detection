# PRD — WiFi Sensing Fall Detection MVP

## 1. 문서 목적
이 문서는 **ESP32-S3 3대와 WiFi CSI(Channel State Information)** 를 활용하여 **실내 1개 공간에서 낙상만 감지하는 MVP** 를 구현하기 위한 제품 요구사항 문서다.

이 프로젝트는 RuView 계열의 아이디어에서 출발하지만, 전체 포즈 추정이나 범용 인체 인식은 범위에서 제외한다. 목표는 다음 두 가지다.

1. **낙상 후보를 실시간으로 감지**한다.
2. **낙상 직후 저움직임 상태를 재확인**하여 오탐을 줄인다.

---

## 2. 제품 개요

### 2.1 문제 정의
고령자, 1인 가구, 병후 회복자, 야간 단독 생활자 환경에서는 실내 낙상 사고를 빠르게 인지하는 것이 중요하다. 카메라는 프라이버시 문제가 있고, 웨어러블은 착용 이탈 문제가 있다. 따라서 **비접촉·비가시광 기반의 WiFi sensing** 으로 낙상을 감지하는 경량 시스템이 필요하다.

### 2.2 비전
카메라 없이, 별도 착용 장치 없이, 특정 방 한 곳에서 안정적으로 낙상 이벤트를 검출하는 데모 가능 수준의 시스템을 만든다.

### 2.3 목표 사용자
- 연구용/졸업작품/PoC를 만드는 개발자
- 특정 방(침실, 거실, 병실)의 낙상 데모를 원하는 팀
- 실사용 전 검증 단계의 프로토타입 사용자

---

## 3. 범위

### 3.1 포함 범위 (In Scope)
- ESP32-S3 3대를 이용한 CSI 수집
- 단일 공간(방 1개) 기준 설치
- 실시간 낙상 감지 파이프라인
- 녹화/수집/라벨링/학습/추론 루프
- 간단한 대시보드 또는 CLI 알림
- 모델 재학습 가능 구조
- 로그, 리플레이, 오프라인 평가

### 3.2 제외 범위 (Out of Scope)
- 전신 포즈 추정
- 사람 수 카운팅
- 호흡/심박 추정
- 다중 방 동시 추적
- 집 전체 범용화
- 의료기기 수준 인증
- 응급 서비스 자동 신고 연동

---

## 4. 성공 기준

### 4.1 제품 성공 기준
MVP 성공은 아래 기준을 만족하는 것으로 정의한다.

1. **특정 1개 방, 특정 설치 위치**에서 안정적으로 동작한다.
2. 낙상과 일반 일상 행동(걷기, 앉기/일어나기, 침대 눕기)을 구분한다.
3. 실시간으로 낙상 후보를 검출한다.
4. 최종 알림은 낙상 직후 저움직임 확인을 포함한다.
5. 수집한 데이터로 모델을 재학습할 수 있다.

### 4.2 정량 지표 초안
- 낙상 후보 탐지 지연: **2초 이내**
- 최종 알림 지연(저움직임 재확인 포함): **8초 이내**
- 실험실/목표 방 기준 민감도(Recall): **90% 이상 목표**
- 데모 환경 기준 정밀도(Precision): **80% 이상 목표**
- 30분 연속 정상 행동 시 거짓 알람: **0~1회 이하 목표**

> 위 지표는 범용 환경이 아닌 **목표 설치 환경** 기준이다.

## 5. 사용자 시나리오

### 시나리오 A — 정상 생활
사용자가 방 안을 걷거나 앉거나 침대에 눕는다. 시스템은 움직임을 인지하더라도 낙상 경보를 울리지 않는다.

### 시나리오 B — 실제 낙상
사용자가 넘어지며 급격한 CSI 변화가 발생한다. 시스템은 낙상 후보를 감지하고, 이후 일정 시간 저움직임이 이어지면 최종 낙상 경보를 발생시킨다.

### 시나리오 C — 오탐 방지
문 열림, 이불 흔들림, 반려동물 이동, 빠른 걷기 등으로 단기 변화가 생겨도 낙상 후 저움직임 조건이 맞지 않으면 알림을 발생시키지 않는다.

## 6. 시스템 환경

### 6.1 하드웨어
- ESP32-S3 노드 3대
- WiFi AP/공유기 1대
- 추론/수집용 PC 또는 미니PC 1대
- 고정 전원
- 가능하면 외부 안테나형 ESP32-S3 보드 권장

### 6.2 설치 환경
- 실내 1개 방
- 노드 3개는 방 가장자리/삼각 배치
- 주요 감지 존: 침대 주변 또는 거실 존
- 노드 높이: 대략 1.2m ~ 1.8m
- 감지 존까지 거리: 대략 2m ~ 4m

## 7. 핵심 요구사항

## 7.1 데이터 수집
시스템은 3개 ESP32-S3 노드에서 CSI를 수집해야 한다.

### 요구사항
- 노드별 raw CSI 또는 가공 직전 수준의 데이터를 받을 수 있어야 한다.
- 노드 식별자, 타임스탬프, RSSI, 채널, subcarrier 관련 정보가 포함되어야 한다.
- 세션 단위 녹화(start/stop)가 가능해야 한다.
- 수집 데이터는 리플레이 가능한 형식으로 저장되어야 한다.

### 수용 기준
- 수집 프로세스 시작 후 3개 노드 데이터가 정상적으로 유입된다.
- 저장 파일만으로 동일 세션 재생이 가능하다.

## 7.2 전처리
시스템은 추론 전 CSI 전처리를 수행해야 한다.

### 요구사항
- amplitude / phase 분리
- phase unwrap
- 이상치 제거
- smoothing 및 drift 제거
- 윈도우 생성
- 노드별 feature 결합

### 수용 기준
- 원본 없이도 전처리 산출물만으로 학습이 가능하다.
- 파이프라인이 배치/실시간 모드에서 동일 로직을 공유한다.

## 7.3 이벤트 분류
시스템은 최소한 아래 이벤트를 분리할 수 있어야 한다.
- empty
- normal_walk
- sit_or_stand_transition
- lie_down_intentional
- fall

단, 운영용 최종 출력은 아래 두 가지면 충분하다.
- non_fall
- fall

### 수용 기준
- 모델 출력은 확률 또는 score 형태를 제공한다.
- 추론 결과에 confidence가 포함된다.

## 7.4 2단계 낙상 판정
시스템은 단일 순간 score만으로 최종 낙상을 확정하지 않는다.

### 1단계
- 급격한 변화 또는 모델 기반 fall probability 상승을 낙상 후보로 본다.

### 2단계
- 후보 이후 5~10초 동안 저움직임 또는 낮은 motion energy 상태를 확인한다.

### 수용 기준
- 후보와 확정 알림이 구분되어 로그에 남는다.
- 저움직임 조건이 충족되지 않으면 final fall로 승격되지 않는다.

## 7.5 실시간 알림
시스템은 실시간 알림 인터페이스를 제공해야 한다.

### 요구사항
- CLI 로그 또는 간단한 웹 대시보드
- 상태: idle / candidate / confirmed
- 이벤트 시각 기록
- 노드 연결 상태 표시

### 수용 기준
- 정상 실행 중 현재 상태를 확인할 수 있다.
- confirmed fall 발생 시 사용자에게 눈에 띄는 경고가 표시된다.

## 7.6 모델 관리
시스템은 모델 재학습과 교체가 쉬워야 한다.

### 요구사항
- 학습 스크립트 분리
- 모델 아티팩트 버전 관리
- 평가 결과 저장
- threshold 설정 파일화

### 수용 기준
- 새 데이터셋으로 재학습 후 추론 프로세스에서 모델 교체가 가능하다.

---

## 8. 비기능 요구사항

### 8.1 신뢰성
- 노드 1개 데이터가 일시적으로 누락되어도 전체 프로세스가 죽지 않아야 한다.
- 데이터 손실, 패킷 누락, 일시적인 disconnect에 대한 로그가 남아야 한다.

### 8.2 확장성
- 향후 노드 수 2~6대로 일반화 가능한 구조여야 한다.
- feature 추출기와 모델을 교체 가능한 모듈 구조여야 한다.

### 8.3 유지보수성
- collector / preprocessing / training / inference / app 계층 분리
- 설정은 코드 하드코딩보다 config 파일 우선
- 실험 로그와 결과 폴더 구조를 일관되게 유지

### 8.4 관측 가능성
- 연결 상태
- 수집 FPS 또는 수집률
- 이벤트 수
- 최근 경고 수
- 노드별 데이터 품질 지표

---

## 9. 시스템 아키텍처

```text
ESP32-S3 x3
   -> UDP/Socket CSI stream
   -> Collector
   -> Preprocessing
   -> Feature extraction / windowing
   -> Model inference
   -> Rule-based confirmation (post-fall inactivity)
   -> Alert UI + logs + saved events
```

## 13. 리스크
